## 1. Functional Requirements

### FR-38: In-App Notifications {: #fr-38 }

!!! success "Acceptance Criteria"
    1. :material-check-all:{ title="Ubiquitous" } The system shall display a **notification bell** in the application header with an unread count badge.
    2. :material-check-all:{ title="Ubiquitous" } The system shall support the following notification types:

        | Type | Trigger |
        |------|---------|
        | `MEAL_PLAN_PUBLISHED` | A user publishes a weekly meal plan |
        | `MEAL_REMINDER` | Generated by the reminder scheduler before a meal's preparation time |
        | `MEAL_SUGGESTION_MADE` | A member changes their assigned meal |
        | `NEW_RECIPE_COMMENT` | A household member comments on a recipe |
        | `NEW_MESSAGE` | A user sends a direct message |

    3. :material-check-all:{ title="Ubiquitous" } The system shall store each notification with a `type`, `title`, `message`, `reference_id`, and `reference_type` for polymorphic linking to the source entity.
    4. :material-play-circle:{ title="Event-driven" } When a user clicks a notification, the system shall mark it as read and navigate to the referenced entity.
    5. :material-check-all:{ title="Ubiquitous" } The system shall poll for new notifications via TanStack Query at a regular interval.
    6. :material-play-circle:{ title="Event-driven" } When a user views the notifications list, the system shall display notifications sorted by `created_at` descending.

!!! quote "Rationale"
    **So That** I stay informed about household activity, I want to receive in-app notifications for important events.

### FR-39: User Messaging {: #fr-39 }

!!! success "Acceptance Criteria"
    1. :material-check-all:{ title="Ubiquitous" } The system shall allow any household member to send a direct message to another household member.
    2. :material-check-all:{ title="Ubiquitous" } The system shall track the `read` status of each message.
    3. :material-play-circle:{ title="Event-driven" } When a message is sent, the system shall create a `NEW_MESSAGE` notification for the recipient.
    4. :material-check-all:{ title="Ubiquitous" } The system shall display messages grouped by conversation (per user pair), sorted by `created_at` ascending within a conversation.

!!! quote "Rationale"
    **So That** family members can communicate within the app, I want to send and receive messages.
