    Tasks: Shopping List Module

    1. Overview

    Implement a comprehensive shopping assistant module that allows household members to manage categories, items, stores, and collaborative shopping lists, complete with loyalty cards and coupon tracking.

    Status Legend

    • Completed
    • In Progress
    • Planned

    2. User Stories

    US-1: Master Data - Categories & Items

    Story Definition

    As a household member, I want to manage my own categories and items, so that I can organize my shopping needs according to my preferences.

    • Persona: Full-stack Engineer
    • Estimate: 6 hours
    • Dependencies: None

    Validates Requirements

    • FR-5: Category & Item Management

    Acceptance Criteria

    1. CRUD operations for Shopping Categories (Name, Description, Icon).
    2. CRUD operations for Shopping Items (Name, Photo, Category).
    3. Every item is linked to a category.
    4. If a category name is not unique, then the system shall return a validation error.

    Implementation Tasks

    • [x] Create Liquibase migration for shopping_categories and shopping_items.
    • [x] Implement JPA Entities and Repositories for Category and Item.
    • [x] Implement REST Controllers with HATEOAS support.
    • [x] Build React management pages using Mantine.

    US-2: Store Management - Loyalty & Coupons

    Story Definition

    As a household member, I want to manage stores, loyalty cards, and coupons, so that I can save money and shop efficiently.

    • Persona: Full-stack Engineer
    • Estimate: 8 hours
    • Dependencies: US-1

    Validates Requirements

    • FR-6: Store Management
    • FR-12: Loyalty Cards
    • FR-13: Store Coupons

    Acceptance Criteria

    1. CRUD for Stores (Name, Description, Icon, Photo).
    2. Add/Display Loyalty Cards with QR/Code 128 rendering.
    3. Manage Coupons with due dates, photos, and "Used" status.
    4. While a loyalty card exists, the system shall share it with the entire household.

    Implementation Tasks

    • [x] Create Liquibase migration for shopping_stores, loyalty_cards, and coupons.
    • [x] Implement JPA Entities and HATEOAS Controllers.
    • [x] Integrate react-barcode and qrcode.react libraries.
    • [x] Build Store details view with nested sections.

    US-3: Shopping Lists - Planning & Suggestions

    Story Definition

    As a household member, I want to create and plan shared shopping lists with price suggestions, so that my family is synchronized on what to buy.

    • Persona: Full-stack Engineer
    • Estimate: 8 hours
    • Dependencies: US-2

    Validates Requirements

    • FR-7: Shopping List Collaboration
    • FR-8: List Items & Quantities
    • FR-9: Price History & Suggestions

    Acceptance Criteria

    1. Create shared shopping lists with a "Completed" status.
    2. Add items to lists with quantity, unit (L, kg, etc.), and optional price.
    3. Where a user adds an item, the system shall suggest the last price paid at the selected store.
    4. Lists shall display the user who created them.

    Implementation Tasks

    • [x] Create Liquibase migration for shopping_lists and shopping_list_items.
    • [x] Implement Price Suggestion logic in ShoppingService.
    • [x] Build "Create/Edit List" UI with searchable item selection.
    • [x] Implement real-time list sharing via TanStack Query.

    US-4: Shopping Mode - In-Store Experience

    Story Definition

    As a shopper, I want to check off items in real-time while at the store, so that I don't miss anything and others know what was bought.

    • Persona: Frontend Engineer
    • Estimate: 6 hours
    • Dependencies: US-3

    Validates Requirements

    • FR-10: In-Store Progress Tracking
    • NFR-3: Reliability & Sync

    Acceptance Criteria

    1. When a user marks an item as "Bought", the system shall move it to the bottom with a strikethrough.
    2. The interface shall be mobile-optimized with a minimum 44x44px tap target.
    3. While offline, the system shall support viewing and checking items.

    Implementation Tasks

    • [x] Implement "Shopping Mode" UI optimized for mobile devices.
    • [x] Implement optimistic updates for check-off actions.
    • [x] Add completed_at timestamp logic for list finalization.

    US-5: Global Maintenance & Warnings

    Story Definition

    As a system user, I want to be warned about expiring coupons and have old data purged, so that the app remains relevant and fast.

    • Persona: Full-stack Engineer
    • Estimate: 4 hours
    • Dependencies: US-2, US-4

    Validates Requirements

    • FR-11: Automatic Data Retention
    • FR-15: Expiration Warning Panel

    Acceptance Criteria

    1. While a coupon is due within 4 days, the system shall display it in the "Expiring Coupons" dashboard panel.
    2. The side navigation shall use a nested "Shopping" menu for all sub-features.
    3. When the daily retention task runs, the system shall delete lists older than 3 months.

    Implementation Tasks

    • [x] Implement Spring @Scheduled task for 3-month data retention.
    • [x] Build the "Expiring Coupons" widget for the Dashboard.
    • [x] Refactor global navigation to use nested structures.

    3. Definition of Done (DoD)

    Quality Checklist

    • [x] Code follows project conventions (Java 25, React 19, Spock).
    • [x] All unit and integration tests (TS-13 to TS-17) are passing.
    • [x] RFC 7807 compliance for all new shopping APIs.
    • [x] Traceability between requirements, design, and implementation is maintained.
