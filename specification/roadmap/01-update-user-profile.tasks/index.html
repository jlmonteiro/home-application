    Tasks: User Profile Updates

    1. Overview

    The goal of this epic is to enable authenticated users to update their profile information and to provide a detailed "Quick View" of this information in the application header.

    Status Legend

    • Completed
    • In Progress
    • Planned

    2. User Stories

    US-1: Backend - Implement Profile Update API

    Story Definition

    As a home user, I want to securely update my profile via a REST API, so that my social and contact details are stored and shared within the application.

    • Persona: Backend Engineer
    • Estimate: 4 hours
    • Dependencies: None

    Validates Requirements

    • FR-4: Update User Profile
    • NFR-1: Security (Zero Trust)

    Acceptance Criteria

    1. PUT /api/user/me endpoint exists and is secured by Spring Security.
    2. Endpoint accepts a JSON body matching the UserProfileUpdateRequest schema.
    3. Returns 200 OK with the updated HATEOAS-compliant user profile.
    4. If invalid phone numbers or social links are provided, then the system shall return 400 Bad Request with RFC 7807 details.
    5. The system shall prevent updates to email, firstName, and lastName.
    6. If the database is unreachable, then the system shall return 503 Service Unavailable.

    Implementation Tasks

    • [x] Add putUserProfile method to UserProfileController.
    • [x] Add updateProfile method to UserProfileService.
    • [x] Ensure UserProfileDTO validation annotations are correctly triggered.
    • [x] Implement UserProfileUpdateSpec.groovy integration test.

    US-2: Frontend - Implement Profile Edit UI

    Story Definition

    As a home user, I want an intuitive interface to edit my profile, so that I can easily update my details without technical knowledge.

    • Persona: Frontend Engineer
    • Estimate: 4 hours
    • Dependencies: US-1

    Validates Requirements

    • FR-4: Update User Profile

    Acceptance Criteria

    1. When the user selects the profile menu, the system shall navigate to the Profile Edit page.
    2. The system shall display existing profile data with email, firstName, and lastName marked as read-only.
    3. Where a user provides a local file, the system shall upload it as a profile photo.
    4. When a user enters data, the system shall provide real-time validation for phone and social URLs.
    5. The system shall show a success notification upon a successful profile update.

    Implementation Tasks

    • [x] Add updateUserProfile function to src/services/api.ts.
    • [x] Create ProfilePage.tsx using useForm from @mantine/form.
    • [x] Implement useMutation for profile updates.
    • [x] Add routing for the new Profile page in App.tsx.

    US-3: Frontend - Enhance Profile Dropdown UI

    Story Definition

    As a home user, I want to see my contact and social info in the header menu, so that I can quickly access these details.

    • Persona: Frontend Engineer
    • Estimate: 2 hours
    • Dependencies: US-1

    Validates Requirements

    • FR-3: User Profile Quick View

    Acceptance Criteria

    1. While data is available, the system shall display the phone number and social icons in the dropdown.
    2. When a user clicks a social icon, the system shall open the link in a new tab.
    3. The system shall provide a link to "View/Edit Profile" within the dropdown menu.
    4. Where profile data is missing, the system shall hide the corresponding sections in the dropdown.

    Implementation Tasks

    • [x] Update Layout.tsx with conditional rendering logic.
    • [x] Integrate Tabler icons into the menu.
    • [x] Link the menu item to the /profile route.
