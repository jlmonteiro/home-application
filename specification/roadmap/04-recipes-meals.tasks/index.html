    Tasks: Recipes & Meals Module

    1. Overview

    Implement a comprehensive recipes and meal planning module that allows household members to manage a shared cookbook, plan weekly meals with per-member assignments, and export meal ingredients to shopping lists. Includes an in-app notification and messaging system.

    Status Legend

    • Completed
    • In Progress
    • Planned

    2. User Stories

    US-1: Recipe CRUD

    Story Definition

    As a household member, I want to create, view, edit, and delete recipes, so that our family can build a shared cookbook.

    • Persona: Full-stack Engineer
    • Estimate: 8 hours
    • Dependencies: None

    Validates Requirements

    • FR-23: Recipe Management

    Acceptance Criteria

    1. CRUD operations for Recipes (name, description in markdown, servings, source link, video link, prep time).
    2. The system shall display the recipe creator's name.
    3. If a recipe is referenced by a meal plan, then the system shall prevent deletion.

    Implementation Tasks

    • [x] Create Liquibase migration for recipes schema and recipes table.
    • [x] Implement JPA Entity, Repository, Service, and DTO for Recipe.
    • [x] Implement REST Controller with HATEOAS support.
    • [x] Build React RecipesListPage, RecipeDetailPage, and RecipeFormPage.

    US-2: Recipe Photos & Dynamic Labels

    Story Definition

    As a household member, I want to add photos and labels to my recipes, so that they are visually appealing and easy to find.

    • Persona: Full-stack Engineer
    • Estimate: 6 hours
    • Dependencies: US-1

    Validates Requirements

    • FR-24: Recipe Photos
    • FR-23: Recipe Management (labels)

    Acceptance Criteria

    1. Upload one or more Base64-encoded photos per recipe.
    2. Designate a default photo for list views.
    3. When a user types a new label, the system shall create it automatically.
    4. When a label is orphaned, the system shall delete it.

    Implementation Tasks

    • [x] Create Liquibase migration for recipe_photos, labels, and recipe_labels tables.
    • [x] Implement entities, repositories, and services for photos and labels.
    • [x] Implement photo upload/delete and default selection endpoints.
    • [x] Implement label autocomplete endpoint (GET /labels?q=).
    • [x] Build photo gallery and label tag input components.

    US-3: Ingredients & Nutrition Data

    Story Definition

    As a household member, I want to add ingredients from the shopping catalog and see nutrition totals, so that I know what I need and can track dietary intake.

    • Persona: Full-stack Engineer
    • Estimate: 6 hours
    • Dependencies: US-1

    Validates Requirements

    • FR-25: Recipe Ingredients
    • FR-26: Nutrition Data

    Acceptance Criteria

    1. Add ingredients by selecting items from shopping.shopping_items with quantity and unit (KG, G, L, ML, PACK, UNIT).
    2. When the recipe detail is requested, the system shall calculate nutrition totals on-the-fly from flexible key-value-unit nutrition entries.
    3. If an ingredient has no nutrition entries, the system shall exclude it from totals and indicate the gap.

    Implementation Tasks

    • [x] Create Liquibase migration for recipe_ingredients and nutrition_entries tables.
    • [x] Implement entities, repositories, and services.
    • [x] Implement on-the-fly nutrition calculation in RecipeService (sum entries grouped by nutrient key × ingredient quantity).
    • [x] Implement ingredient CRUD and nutrition GET/PUT endpoints.
    • [x] Build ingredient selector and nutrition summary UI.

    US-4: Preparation Steps with Drag-and-Drop

    Story Definition

    As a household member, I want to add ordered preparation steps and reorder them via drag-and-drop, so that anyone can follow the recipe.

    • Persona: Full-stack Engineer
    • Estimate: 5 hours
    • Dependencies: US-1

    Validates Requirements

    • FR-27: Preparation Steps

    Acceptance Criteria

    1. Add steps with markdown instruction and optional time_minutes.
    2. Steps are persisted and displayed in sort_order.
    3. When steps are reordered, the system shall update all sort_order values in a single transaction.

    Implementation Tasks

    • [x] Create Liquibase migration for recipe_steps table.
    • [x] Implement entity, repository, and service with reorder logic.
    • [x] Implement step CRUD and PUT /recipes/{id}/steps/reorder endpoint.
    • [x] Build step list with @dnd-kit/core drag-and-drop.

    US-5: Comments & Ratings

    Story Definition

    As a household member, I want to comment on and rate recipes, so that we can share feedback and identify favorites.

    • Persona: Full-stack Engineer
    • Estimate: 5 hours
    • Dependencies: US-1

    Validates Requirements

    • FR-28: Recipe Comments
    • FR-29: Recipe Ratings

    Acceptance Criteria

    1. Any household member can add comments.
    2. 1-5 star rating per user (unique constraint).
    3. Display average rating with expandable individual votes (who, score, when).
    4. When a comment is added, create a NEW_RECIPE_COMMENT notification.

    Implementation Tasks

    • [x] Create Liquibase migration for recipe_comments and recipe_ratings tables.
    • [x] Implement entities, repositories, and services.
    • [x] Implement comment and rating endpoints.
    • [x] Build comments section and star rating component with expandable votes.

    US-6: Meal Time Configuration

    Story Definition

    As a household member, I want to configure meal times with different schedules per day, so that the planner reflects our actual routine.

    • Persona: Full-stack Engineer
    • Estimate: 4 hours
    • Dependencies: None

    Validates Requirements

    • FR-30: Meal Time Configuration

    Acceptance Criteria

    1. CRUD for meal times (Breakfast, Lunch, Dinner, etc.).
    2. Configure different times per day of the week (ISO 8601: 1=Monday, 7=Sunday).

    Implementation Tasks

    • [x] Create Liquibase migration for meals schema, meal_times and meal_time_schedules tables.
    • [x] Implement entities, repositories, and services.
    • [x] Implement CRUD endpoints for meal times with schedules.
    • [x] Build MealTimesConfigPage with per-day time inputs.

    US-7: Weekly Meal Planner

    Story Definition

    As a household member, I want to plan weekly meals by assigning recipes to meal times and members, so that our family knows what to eat each day.

    • Persona: Full-stack Engineer
    • Estimate: 10 hours
    • Dependencies: US-1, US-6

    Validates Requirements

    • FR-31: Weekly Meal Plan

    Acceptance Criteria

    1. Create weekly plans (Monday–Sunday, unique week_start_date).
    2. Assign one or more recipes per meal entry (multi-recipe meals).
    3. Where a recipe is assigned to specific members, record the user_id. Null means for everyone.
    4. When a user marks an entry as "done", update the is_done flag.

    Implementation Tasks

    • [x] Create Liquibase migration for meal_plans, meal_plan_entries, and meal_plan_entry_recipes tables.
    • [x] Implement entities, repositories, and services.
    • [x] Implement CRUD endpoints for plans, entries, and recipe assignments.
    • [x] Build MealPlannerPage with weekly grid (7 columns × N meal times).

    US-8: Meal Plan Approval, Thumbs Up/Down & Reminders

    Story Definition

    As a household member, I want to review meal plans, give feedback, and receive reminders, so that everyone has a say and nobody forgets to cook.

    • Persona: Full-stack Engineer
    • Estimate: 8 hours
    • Dependencies: US-7, US-10

    Validates Requirements

    • FR-32: Meal Plan Approval Workflow
    • FR-33: Meal Preparation Reminders
    • FR-37: Meal Thumbs Up/Down

    Acceptance Criteria

    1. When "Notify Household" is clicked, set status to PUBLISHED and create notifications.
    2. When a member accepts, set status to ACCEPTED.
    3. When a member suggests a change, auto-replace their assignment and set status to CHANGED.
    4. Where a meal is done, allow thumbs up/down votes.
    5. Reminder scheduler runs every 15 minutes and creates MEAL_REMINDER notifications.

    Implementation Tasks

    • [x] Create Liquibase migration for meal_plan_votes table.
    • [x] Implement member response and vote services.
    • [x] Implement notify, respond, vote, and done endpoints.
    • [x] Implement MealReminderScheduler (every 15 min).
    • [x] Build approval UI with accept/suggest actions and thumbs up/down buttons.

    US-9: Shopping List Integration

    Story Definition

    As a household member, I want to export meal ingredients to a shopping list, so that I can shop for what I need to cook.

    • Persona: Full-stack Engineer
    • Estimate: 6 hours
    • Dependencies: US-3, US-7

    Validates Requirements

    • FR-36: Shopping List Integration

    Acceptance Criteria

    1. When export is requested, generate a merge preview (new items + existing items with quantity changes).
    2. When the user confirms, add items to the list or create a new one.
    3. No store assigned to new items unless already in the list.
    4. If an item already exists, increase quantity rather than creating a duplicate.

    Implementation Tasks

    • [x] Implement export preview and merge logic in MealPlanService.
    • [x] Implement preview and export endpoints.
    • [x] Build merge confirmation modal with item summary.

    US-10: Notifications & Messaging

    Story Definition

    As a household member, I want to receive notifications and send messages, so that I stay informed and can communicate within the app.

    • Persona: Full-stack Engineer
    • Estimate: 8 hours
    • Dependencies: None

    Validates Requirements

    • FR-38: In-App Notifications
    • FR-39: User Messaging

    Acceptance Criteria

    1. Notification bell in header with unread count badge.
    2. Typed notifications (MEAL_PLAN_PUBLISHED, MEAL_REMINDER, MEAL_SUGGESTION_MADE, NEW_RECIPE_COMMENT, NEW_MESSAGE).
    3. When a notification is clicked, mark as read and navigate to the referenced entity.
    4. Direct messaging between household members with read status.

    Implementation Tasks

    • [x] Create Liquibase migration for notifications schema, notifications and messages tables.
    • [x] Implement entities, repositories, and services.
    • [x] Implement notification and messaging endpoints.
    • [x] Build NotificationBell component, NotificationsPage, and messaging UI.
